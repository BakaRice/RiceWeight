# 体重趋势图页面需求拆解

## 背景

在增加趋势页之前，RiceWeight 已经可以记录体重和查看历史记录，但还不能直观地观察体重随时间的变化。

本次计划新增一个体重趋势页面，使用折线图展示历史记录。项目以学习 SwiftUI 为主要目标，因此首版只实现容易理解的核心功能，不一次加入复杂筛选和交互。

## 实现状态

首版目标已完成。后续继续学习图表时，可以从“暂不实现”一节中选择一个小功能逐步增加。

## 首版目标

新增一个趋势页面，展示：

- 按日期升序排列的体重折线图
- 按天展示的横轴刻度
- 每一条体重记录对应的数据点
- 固定目标体重 `75.0 kg` 对应的水平参考线
- 没有记录时的空状态提示

## 实现步骤

### 第一步：新增趋势页面

新建 `RiceWeight/WeightChartView.swift`。

页面负责读取体重记录，并根据是否存在数据展示图表或空状态。

在首页增加进入趋势页的入口，例如在工具栏中增加一个图表按钮，并通过 `NavigationLink` 跳转。

涉及知识点：

- `NavigationStack`
- `NavigationLink`
- 将一个页面拆分为独立的 SwiftUI `View`

### 第二步：展示折线图和数据点

使用 Apple 官方的 `Charts` 框架，不引入第三方依赖。

图表中组合：

- `LineMark`：连接不同日期的体重记录
- `PointMark`：标出每一次记录

历史列表适合按日期倒序展示，让最新数据出现在最上方。折线图适合按日期升序展示，让折线从左向右表示时间变化。两处可以使用不同的排序方式。

涉及知识点：

- `import Charts`
- `Chart`
- `LineMark`
- `PointMark`
- 数组排序

### 第三步：增加目标体重参考线

使用 `RuleMark` 增加一条 `75.0 kg` 的水平虚线，并使用与折线不同的颜色。

目标体重目前仍然保持固定，不在本次任务中增加设置功能。

涉及知识点：

- `RuleMark`
- 在同一张图表中组合多种标记
- 使用样式区分不同类型的数据

### 第四步：补充国际化文案

新增用户可见文案时，在 `RiceWeight/L10n.swift` 中增加语义明确的访问入口，并在 `RiceWeight/Localizable.xcstrings` 中补充日语、简体中文和英文翻译。

预计需要的文案：

| Key | 用途 |
| --- | --- |
| `chart.title` | 趋势页面标题 |
| `chart.empty` | 没有体重记录时的提示 |
| `chart.target` | 目标体重参考线标签 |
| `action.showChart` | 首页进入趋势页按钮的无障碍说明 |

涉及知识点：

- 使用 `L10n` 集中管理文案
- 为图标按钮增加无障碍说明
- 在 String Catalog 中维护多语言翻译

## 首版边界情况

- 没有记录：展示空状态提示，不展示空白图表。
- 只有一条记录：展示一个数据点。折线可能暂时不可见，这是正常情况。
- 有多条记录：按照记录日期升序连接数据点，横轴以天为单位展示。
- 存在已软删除记录：不在趋势图中展示。

图表横轴按天展示只是界面层的选择。`WeightRecord.measuredAt` 仍然使用 `Date` 保存时间点。
新增记录时默认使用当前时间，用户也可以在录入页通过独立的两行分别修改日期、小时和分钟。

- 动画效果
- 修改目标体重
- 折线图以外的其他图表

## 验收标准

1. 首页可以进入趋势页面。
2. 没有历史记录时，趋势页面展示本地化空状态提示。
3. 新增一条记录后，趋势页面展示对应的数据点。
4. 新增多条不同日期的记录后，数据点按日期从左向右连接。
5. 图表中可以看到 `75.0 kg` 的目标体重参考线。
6. 软删除一条记录后，对应的数据点不再展示。
7. 页面新增文案支持日语、简体中文和英文。
8. 修改完成后运行可用的 Xcode 构建检查。
